# C ABI
In computer programming, the C ABI (Application Binary Interface) is a low-level set of rules that defines how compiled C code interacts with other binary components, such as libraries or the operating system. 
While an API (Application Programming Interface) tells a developer how to write source code (e.g., function names and types in a header file), the ABI tells the computer how to execute that code after it has been turned into a binary. 
## Core Components of the C ABI
The C ABI covers several critical technical areas to ensure different programs can "talk" to each other: 

* Calling Conventions: Rules for how functions receive arguments (e.g., which CPU registers to use vs. using the stack) and how they return values.
* Data Layout: Standardizes the size and alignment of basic types like int or float, and how members are arranged inside a struct (including where padding is added).
* Symbol Mangling: Defines how function names are stored in the binary file so the linker can find them. In C, this is usually simple, but it is vital for compatibility with other languages like C++.
* System Calls: Specifies the exact method for a program to request services from the operating system kernel, such as opening a file or allocating memory. 

## Why It Is Important

* Stability: On platforms like Linux or Windows, the C ABI is kept extremely stable. This allows a program compiled years ago to run on a modern system without needing to be recompiled.
* Interoperability: The C ABI is the "lingua franca" of software. Most other languages (Rust, Python, Java, etc.) use it to call C libraries or allow themselves to be called by others through a Foreign Function Interface (FFI).
* Binary Compatibility: It allows libraries from different vendors or compiled with different compilers to be linked together into a single working application. 

## Platform Specifics
There is no single "universal" C ABI. Each combination of hardware (CPU architecture) and operating system has its own, such as: 

* System V AMD64 ABI: Used by Linux and macOS on x86-64 processors.
* Microsoft x64 Calling Convention: Used by Windows on x86-64 processors.
* ARM EABI: A specialized version for embedded systems using ARM processors.
